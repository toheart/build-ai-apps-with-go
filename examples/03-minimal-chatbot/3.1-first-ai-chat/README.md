# 3.1 对应代码说明

这个目录存放 `3.1 先让模型开口：跑通第一个 AI 对话` 的最小可运行代码。

## 文件说明

- `main.go`：程序入口，读取配置并完成一次最小模型调用，同时支持 `sync` 与 `stream` 两种响应模式
- `go.mod`：Go 模块定义
- `.env.example`：示例环境变量

## 运行方式

1. 复制配置文件

```bash
cp .env.example .env
```

2. 修改 `.env` 中的模型服务配置

3. 运行程序

```bash
go run .
```

如果你想显式选择模式，可以这样运行：

```bash
go run . -mode sync
go run . -mode stream
```

如果没有传 `-mode`，程序会优先读取 `OPENAI_RESPONSE_MODE`；如果环境变量也没设置，则默认使用 `stream`。

## 本阶段能力

这一版代码只做下面几件事：

- 接收一条用户输入
- 调用一次模型接口
- 支持 `sync` 模式一次性读取完整回复
- 支持 `stream` 模式持续接收模型回复并实时打印

## 本阶段边界

这一版故意不包含以下内容：

- 多轮对话消息历史
- 工具调用
- 前端页面
- 完整日志系统

这样做的目的，是先让读者把第一条真实模型调用链路跑通，再进入下一步。
